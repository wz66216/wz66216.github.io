# 音乐播放器设计方案 (Music Player Design Spec)

## 目标 (Objective)
在博客右下角添加一个基于酷狗音乐 API 的自定义、原生 (Vanilla JS) 音乐播放器。
它需要具备高颜值、支持自由拖动、并完美契合博客现有的核心视觉语言：**Swiss Grid × Chess**（黑白极简、棋盘格、国际象棋元素）。

## 探索方案评估 (Exploration Evaluation)

在 Phase 2 中，我们探讨了 3 种潜在的现代播放器 UI 架构。结合目前博客的“纯黑白 + 棋盘格 + 极简主义”核心主题，以下是具体评估：

### 1. 灵动岛风格 (Dynamic Island)
- **视觉特征**：顶部或底部的胶囊状悬浮条，点击后平滑展开为玻璃态卡片，显示歌单和滚动歌词。
- **匹配度**：中高。极简的胶囊形态能很好地融入博客，但玻璃拟物化 (Glassmorphism) 可能与目前的粗犷、黑白分明的 Swiss Grid 风格稍显冲突。

### 2. 极简黑胶/唱片机风格 (Minimalist Floating Vinyl) - 🏆 推荐方案 (Recommended)
- **视觉特征**：默认状态下是一个圆形的黑白唱片（或国际象棋的棋子图腾），唱片在播放时匀速旋转。鼠标悬停或点击时，从唱片侧边滑出一条极简的黑白控制面板（包含播放控制、进度条、歌词信息）。
- **匹配度**：极高。圆形唱片完美契合国际象棋的底座几何形态；黑白极简控制条可以通过 CSS 变量严丝合缝地适配 `light` 和 `dark` 主题。无缝对接现有的 CSS 变量系统。
- **拖拽体验**：用户可以直接拖动这个“圆形黑胶”或“棋子图标”，将其放置在屏幕的任何角落，体验自然顺滑。

### 3. 赛博朋克终端 (Cyberpunk Terminal)
- **视觉特征**：带有终端代码字体、复古窗口边框和音频频谱。
- **匹配度**：低。因为博客核心设计规范明确为 “Swiss Grid × Chess”，而不是 Cyberpunk。这套方案视觉过于喧宾夺主，会破坏博客本身优雅阅读的氛围。

---

## 最终实现计划 (Implementation Plan) -> Phase 5 Handoff

基于评估，我们决定采用 **极简黑胶风格 (Minimalist Floating Vinyl) 结合 国际象棋元素**：

1. **结构 (DOM)**：
   - 默认折叠态：一个悬浮的圆形黑胶/光盘（中心可以使用 `data:image` 或者歌曲封面），带一圈深色/浅色的边框。
   - 展开态：一个带有锐利边角的矩形面板（无圆角，贴合 Swiss Grid 设计），内部显示当前曲目、艺术家、进度条和播放按钮。
2. **样式 (CSS)**：
   - 完全抛弃 `box-shadow` 带来的复杂拟物感，改用锋利的 `border`（符合现有的 `border: 2px solid var(--color-border)`）。
   - 按钮图标使用 Unicode，如播放 `▶`、暂停 `⏸`、上一首 `⏮`。
   - 进度条采用最简单的黑白填充。
3. **数据接入 (Data)**：
   - 维持现有的第三方 Meting 接口：`https://api.injahow.cn/meting/?server=kugou&type=playlist&id=[ID]`。
   - 初始化时获取播放列表，并准备好原生 `Audio` 对象逻辑。
4. **交互 (Interaction)**：
   - 保留拖拽逻辑。
   - 悬停/点击唱片图标时，展开播放面板（宽度从 `0` 过渡到 `250px`）。
